ItsNat provides five communication modes for events:

  1. AJAX Synchronous

    In this mode the browser is locked until the event sent to the server returns.

  2. AJAX Asynchronous

    In this mode the event is sent asynchronously, the browser can fire and send to the server new events.

  3. AJAX Asynchronous-Hold

    Is provided by ItsNat, in this mode the event is sent to the server asynchronously (the browser is not locked) but any new event fired to be sent to the server is queued as a FIFO, when the first event returns the first queued event is sent to the server (applying its synchronization rule). This mode is very interesting because offers synchronous/predictable behavior with no browser locks.
  4. SCRIPT

    In this mode auxiliary SCRIPT elements are used for events, this type of communication is essentially asynchronous.

  5. SCRIPT Hold

    This mode is based on SCRIPT elements but using the same technique of AJAX Asynchronous-Hold to provide a synchronous/predictable behavior with no browser locks.
