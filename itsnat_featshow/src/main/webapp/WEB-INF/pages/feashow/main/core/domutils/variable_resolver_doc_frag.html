ItsNatVariableResolver objects are very useful to render data as markup in an independent way of the concrete markup.

Previous examples used Java based renderers tied to the concrete markup layout, this can be avoided using ItsNatVariableResolvers. An ItsNatVariableResolver uses a "minimalist" expression language, in fact there is no expression language at all, only variables with the form ${someName} can be defined in markup to be resolved using a variable resolver. This example shows how the rendering code can be heavily simplified and independent from the markup.

An ItsNatVariableResolver can resolve the variables declared in a markup fragment using local variables (name/value pairs) and variables defined as attributes in the ItsNatServletRequest, ItsNatDocument, ItsNatSession and ItsNatServletContext depending how it was created (for instance calling ItsNatDocument.createItsNatVariableResolver).

The method ItsNatDocument.createItsNatVariableResolver(boolean disconnected) with disconnected param set to true creates a standalone variable resolver.
