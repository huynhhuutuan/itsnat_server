This example shows how to upload a file using <iframe> and <input type=file> elements.

A component ItsNatHTMLIFrame is used to wrap the <iframe> element working as a factory of the file upload action, managed by HTMLIFrameFileUpload.

When a HTMLIFrameFileUpload is created, a file upload request is sent to the client, ItsNat automatically creates a temporary <form> to send to the <iframe> the file upload request with the file path obtained from the <input type=file> element.

<input type=file> does not need an enclosing form and a name attribute, but this attribute is mandatory in <iframe> and cannot be changed after insertion (this is not an ItsNat limitation it is imposed by browsers and has to do with security).

Because file uploading may be a long task and taken place in a different "page" (an iframe) the thread of the web request uploading the file is not synchronized with the associated ItsNatDocument, any access to the ItsNatDocument or dependent objects must be synchronized before (for instance to cancel the timer).

Of course the <iframe> element may be hidden and can be dynamically inserted when the file upload is requested, and removed when file uploading finishes.
