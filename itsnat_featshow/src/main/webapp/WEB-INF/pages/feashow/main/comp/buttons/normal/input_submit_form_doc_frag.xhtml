In this example the submit button is contained by a form, now a submit sends the form leaving the page. This an opportunity to show how an event can be cancelled in the server calling org.w3c.dom.events.Event.preventDefault().

The cancellation only works in AJAX synchronous mode because the cancellation only is possible inside the client event processing immediately after the event is fired, this is not an ItsNat limitation is a limitation of browsers. To send/receive button clicks in AJAX synchronous mode the call ItsNatComponent.setEventListenerParams(...,CommMode.XHR_SYNC,...) sets the communication mode to AJAX synchronous (by default the Feature Showcase mode is CommMode.XHR_ASYNC_HOLD a deterministic variation of the asynchronous mode).

Some browsers (usually based on old WebKit versions) do not support synchronous AJAX.

The same is applied to <input> buttons with type "reset" inside forms.
