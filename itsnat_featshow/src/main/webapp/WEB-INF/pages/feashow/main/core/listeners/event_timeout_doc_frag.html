A timeout may be specified in any AJAX/SCRIPT event listener. This is the maximum time an asynchronous (or asynchronous-hold) AJAX/SCRIPT request will wait to the server before aborting the request. A negative value specifies no timeout.

In synchronous AJAX events the timeout is ignored (a synchronous event cannot be aborted).

Use this feature with care (by default there is no timeout), an aborted request may stop a COMET process, a timer, a remote view/control process etc, and the worst thing client may remain unsynchronized.
