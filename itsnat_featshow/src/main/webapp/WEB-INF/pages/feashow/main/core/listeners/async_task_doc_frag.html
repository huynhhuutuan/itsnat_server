ItsNat automatically locks (synchronizes) the ItsNatDocument object during event processes, this prevents other asynchronous events to modify the server document and dependent objects concurrently, so the server user code may be thread unsafe with no problem.

The drawback of this approach is asynchronous long server tasks, the recommended technique is to execute this long task using a different (or new) thread to avoid locking the server document during long time.

ItsNat helps to notify the client when an asynchronous long task finishes, registering the Runnable object to execute in a new thread calling ClientDocument.addAsynchronousTask, of course using AJAX or SCRIPT elements behind the scenes. When the long task finishes any DOM node tree change or JavaScript added to send to the client (calling ItsNatDocument.addCodeToSend) is returned to the client automatically.
