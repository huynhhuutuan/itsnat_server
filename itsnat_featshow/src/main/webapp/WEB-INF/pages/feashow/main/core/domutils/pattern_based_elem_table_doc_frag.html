The ElementTable utility interface is a pattern based element table, unlike an ElementTableFree DOM row and cell elements of the table have the same tag and are created using a pattern approach. An ElementTable object is created calling an ElementGroupManager.createElementTable method and attached to a DOM element parent of the table.

The original markup of the subtree being associated to the ElementTable must contain one row item element and inside this row and almost a cell element, this row and the first cell will be the row and cell patterns and will be saved internally by the table and used to create new rows and cells.

A Java data object can be rendered as markup inside a table using a ElementTableRenderer if provided (otherwise the default is used). The cell element provided to render the data object as markup is the element parent of the cell (this element is a direct child of the row element) but it can change using a user defined ElementTableStructure.
