In this example the initial page is based on a normal template with state disabled (scripting is disabled) and no processing, this is basically the same as loading a static file. This client page will be sent to the server converted to DOM inside an ItsNatDocument and processed by Java code, in this case the same Java code of the Core example in manual.

One "attached server template" must be registered in server calling the method ItsNatServlet.registerItsNatDocumentTemplateAttachedServer(...) specifying the Java class going to process the client page as if it was a normal template.

Refer to the Reference Manual for a complete description of extreme mashups (and why onclick attributes are needed for MSIE v6).
