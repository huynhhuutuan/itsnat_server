As Internet Explorer (6,7,8) has not W3C mutation events, ItsNat provides alternatives way to synchronize the server DOM when the browser DOM changes valid on MSIE and W3C based.

  • Autosync Attributes:

    This example shows how to automatically update a changed attribute on client in the server.

    Passing a NodeAttributeTransport parameter to the method ItsNatDocument.addEventListener commands ItsNat to transport the current specified client attribute value when the client event is fired, this value is updated automatically in the server, removing or creating the attribute if necessary.

    The NodeAllAttribTransport alternative takes a snapshot of the attributes of the specified element.

  • Autosync the Content (Inner) of a Node:

    Similar to NodeAllAttribTransport the class NodeInnerTransport is used to update the server node content (inner) with the current content of the client counterpart.

  • Autosync a Node Completely

    NodeCompleteTransport is used to update a server node with the current state of the client counterpart, content and attributes. Is the sum of the behavior of NodeAllAttribTransport and NodeInnerTransport classes.

  • Autosync Node Properties

    JavaScript DOM distinguishes between attributes and properties, some attributes have counterpart properties; property and attribute values may be different for instance the attribute/property "value". Java W3C DOM does not distinguish between attributes and properties, properties ever are saved as attributes.

    Similar to NodeAttributeTransport the class NodePropertyTransport is used to update a server node property/attribute with the current value of the specified client property counterpart.
